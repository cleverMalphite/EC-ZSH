## 1. 模块概述

### 1.1 功能定位
本项目实现了一个基于GStreamer的实时视频传输系统，支持：
- MP4文件循环播放作为视频源
- H.264编码与RTP封装
- 自定义传输层（SpeedControl + MRUDP）
- 自适应码率控制
- 实时反馈与丢包统计
- 本地预览与远端接收展示

### 1.2 核心特性
- **零接触启动**：接收端自动启动，无需手动触发
- **深度集成**：GStreamer管线与自定义传输层无缝对接
- **自适应传输**：根据网络状况动态调整编码码率
- **循环播放**：MP4文件播放结束后自动重新开始
- **实时统计**：每秒输出发送/接收速率、丢包率等指标
- **MRUDP传输**：使用MRUDP协议传输视频数据（非RBUDP）

### 1.3 重要说明：传输协议

**本项目的视频数据通过MRUDP协议传输，而非RBUDP！**

虽然代码中存在`isRBUDP`参数，但在视频传输模块中，该参数始终设置为`false`，表示使用MRUDP传输：

```cpp
// 发送端和接收端都使用MRUDP
SendTIDDataWithSpeedControl(destTID, buffer, length, 
                           false, false, false);
                           //              ↑
                           //         isRBUDP=false
```

**MRUDP vs RBUDP：**
- **MRUDP**（当前使用）：多路径可靠UDP，适合实时视频流
  - 支持多路径传输
  - 灵活的可靠性控制
  - 更好的拥塞控制
  - 允许部分丢包以保证实时性
  
- **RBUDP**（未使用）：可靠批量UDP，适合大文件传输
  - 强调完全可靠性
  - 批量传输优化
  - 适合非实时场景

### 2.1 系统层次结构
```
┌─────────────────────────────────────────────────────────────┐
│                        UI层 (Qt)                             │
│  ┌──────────────────┐              ┌──────────────────┐     │
│  │  VideoSender     │              │  VideoReceiver   │     │

│  └──────────────────┘              └──────────────────┘     │
└─────────────────────────────────────────────────────────────┘
                    │                          │
                    ▼                          ▼
┌─────────────────────────────────────────────────────────────┐
│                   GStreamer 管线层                           │
│  ┌──────────────────────────────────────────────────────┐   │
│  │ 发送管线: MP4解码 → H264编码 → RTP封装 → appsink     │   │
│  │ 接收管线: appsrc → RTP解包 → H264解码 → BGR输出      │   │
│  └──────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
                    │                          │
                    ▼                          ▼
┌─────────────────────────────────────────────────────────────┐
│                   传输层 (自定义协议)                         │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  SpeedControl (速率控制 + 数据回调)                   │   │
│  │  └─→ NetCombTransfer (网络组合传输)                  │   │
│  │      └─→ MRUDP (多路径可靠UDP)                       │   │
│  └──────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 数据流向
```
[发送端]
MP4文件 → uridecodebin → videoconvert → videoscale → videorate
  → video/x-raw(640x480@30fps) → tee
     ├─→ [预览分支] videoconvert → BGR → appsink_preview → PlayerWidget
     └─→ [RTP分支] x264enc → rtph264pay → appsink_rtp
          → 添加0x00头 → SendTIDDataWithSpeedControl
          → SpeedControl → NetCombTransfer → MRUDP → 网络

[接收端]
网络 → MRUDP → NetCombTransfer → SpeedControl回调
  → 解析0x00头 → RTP包 → gst_app_src_push_buffer
  → rtpjitterbuffer → rtph264depay → h264parse → avdec_h264
  → videoconvert → BGR → appsink → PlayerWidget
```

### 3.2 Deep Integration 管线构建

#### 3.2.1 管线字符串
```cpp
void VideoSender::startDeepIntegrationPipeline(int destTID) {
    m_destTID = destTID;
    
    // 视频源：MP4文件URI
    const std::string file_uri = to_file_uri(resolve_default_video_file_path());
    
    // 预览分支：BGR格式输出到appsink
    const std::string preview_branch =
        "t. ! queue ! videoconvert ! video/x-raw,format=BGR ! "
        "appsink name=preview_sink drop=true max-buffers=1 sync=true";
    
    // RTP分支：H264编码 + RTP封装
    const std::string rtp_branch =
        "t. ! queue ! "
        "x264enc name=enc tune=zerolatency speed-preset=superfast "
        "bitrate=2500 key-int-max=30 bframes=0 ! "
        "rtph264pay mtu=1200 pt=96 config-interval=1 ! "
        "appsink name=rtp_sink emit-signals=true sync=true "
        "max-buffers=10 drop=true";
    
    // 完整管线
    const std::string pipeline_str =
        "uridecodebin uri=" + file_uri + " ! "
        "videoconvert ! videoscale ! videorate ! "
        "video/x-raw,width=640,height=480,framerate=30/1 ! "
        "tee name=t " + preview_branch + " " + rtp_branch;
    
    // 创建管线
    pipeline = gst_parse_launch(pipeline_str.c_str(), &error);
    
    // 获取关键元素
    encoder = gst_bin_get_by_name(GST_BIN(pipeline), "enc");
    appsink_rtp = gst_bin_get_by_name(GST_BIN(pipeline), "rtp_sink");
    
    // 连接RTP数据回调
    g_signal_connect(appsink_rtp, "new-sample", 
                     G_CALLBACK(on_new_sample_from_sink), this);
    
    // 启动管线
    gst_element_set_state(pipeline, GST_STATE_PLAYING);
    timer_appsink->start(33);  // 启动预览定时器
}
```

#### 3.2.2 关键参数说明

**x264enc 编码器参数：**
- `tune=zerolatency`：零延迟模式，适合实时传输
- `speed-preset=superfast`：快速编码，降低CPU占用
- `bitrate=2500`：初始码率2.5Mbps
- `key-int-max=30`：关键帧间隔30帧（1秒）
- `bframes=0`：禁用B帧，减少延迟

**rtph264pay RTP封装参数：**
- `mtu=1200`：最大传输单元1200字节
- `pt=96`：RTP payload type
- `config-interval=1`：每秒发送一次SPS/PPS

**appsink_rtp 输出参数：**
- `emit-signals=true`：启用信号回调
- `sync=true`：同步时钟
- `max-buffers=10`：最多缓存10个buffer
- `drop=true`：缓存满时丢弃旧数据

### 3.3 RTP数据提取与封包

#### 3.3.1 appsink回调
```cpp
GstFlowReturn VideoSender::on_new_sample_from_sink(
    GstElement *sink, VideoSender *sender) {
    
    if (!sender || sender->m_isStopping.load()) {
        return GST_FLOW_OK;
    }
    
    // 从appsink拉取sample
    GstSample *sample;
    g_signal_emit_by_name(sink, "pull-sample", &sample);
    
    if (sample) {
        GstBuffer *buffer = gst_sample_get_buffer(sample);
        GstMapInfo map;
        
        if (gst_buffer_map(buffer, &map, GST_MAP_READ)) {
            // 将RTP数据注入网络
            sender->handleRtpData(map.data, map.size);
            gst_buffer_unmap(buffer, &map);
        }
        
        gst_sample_unref(sample);
        return GST_FLOW_OK;
    }
    
    return GST_FLOW_ERROR;
}
```

#### 3.3.2 数据封包与发送
```cpp
void VideoSender::handleRtpData(guint8* data, gsize size) {
    if (size <= 0 || m_destTID <= 0 || m_isStopping.load()) {
        return;
    }
    
    // 封包格式：[0x00][RTP payload]
    DWORD totalLen = size + 1;
    std::shared_ptr<BYTE> sendBuf(new BYTE[totalLen], 
                                  std::default_delete<BYTE[]>());
    
    sendBuf.get()[0] = 0x00;  // 视频RTP数据标识
    memcpy(sendBuf.get() + 1, data, size);
    
    // 通过SpeedControl发送
    bool ret = SendTIDDataWithSpeedControl(
        static_cast<DWORD>(m_destTID),
        sendBuf,
        totalLen,
        false,  // isUrgent
        false,  // isSleep
        false   // isRBUDP (使用MRUDP)
    );
    
    // 统计
    if (ret) {
        m_sendPktsWindow += 1;
        m_sendBytesWindow += totalLen;
    } else {
        m_sendFailWindow += 1;
    }
    
    // 每秒输出统计
    const qint64 elapsedMs = m_sendStatClock.elapsed();
    if (elapsedMs >= 1000) {
        const uint64_t bps = (m_sendBytesWindow * 8ULL * 1000ULL) / elapsedMs;
        const uint64_t pps = (m_sendPktsWindow * 1000ULL) / elapsedMs;
        
        std::cout << "[VideoSender] TID=" << getTID()
                  << " -> DEST_TID=" << m_destTID
                  << " send_pps=" << pps
                  << " send_kbps=" << (bps / 1000ULL)
                  << " send_fail=" << m_sendFailWindow
                  << std::endl;
        
        // 重置统计窗口
        m_sendPktsWindow = 0;
        m_sendBytesWindow = 0;
        m_sendFailWindow = 0;
        m_sendStatClock.restart();
    }
}
```

### 3.4 自适应码率控制

#### 3.4.1 反馈包接收
```cpp
bool VideoSender::onSpeedControlRecvData(
    DWORD srcTID,
    const std::shared_ptr<BYTE> &pBuffer,
    DWORD dwLength,
    long int &recvtime,
    long int &fb_send_time) {
    
    if (!pBuffer || dwLength < 13) {
        return true;
    }
    
    const BYTE *typed = pBuffer.get();
    
    // 检查反馈包标识（0x01）
    if (typed[0] != 0x01) {
        return true;
    }
    
    // 解析反馈数据（网络字节序）
    uint32_t recv_bps = 0;      // 接收端速率(bps)
    uint32_t loss_x1000 = 0;    // 丢包率*1000
    uint32_t last_seq = 0;      // 最后序列号
    
    memcpy(&recv_bps, typed + 1, 4);
    memcpy(&loss_x1000, typed + 5, 4);
    memcpy(&last_seq, typed + 9, 4);
    
    recv_bps = ntohl(recv_bps);
    loss_x1000 = ntohl(loss_x1000);
    last_seq = ntohl(last_seq);
    
    // 根据丢包率调整目标码率
    const uint32_t min_bps = 200000;   // 最小200kbps
    const uint32_t max_bps = 8000000;  // 最大8Mbps
    
    uint32_t target_bps = recv_bps;
    if (loss_x1000 > 0 && loss_x1000 <= 1000) {
        // 目标码率 = 接收速率 * (1 - 丢包率)
        target_bps = static_cast<uint32_t>(
            (static_cast<uint64_t>(recv_bps) * (1000ULL - loss_x1000)) 
            / 1000ULL
        );
    }
    
    // 限制范围
    if (target_bps < min_bps) target_bps = min_bps;
    if (target_bps > max_bps) target_bps = max_bps;
    
    const uint32_t target_kbps = target_bps / 1000;
    
    // 变化超过50kbps才更新（避免频繁调整）
    if (m_targetBitrateKbps == 0 ||
        abs(target_kbps - m_targetBitrateKbps) >= 50) {
        
        m_targetBitrateKbps = target_kbps;
        
        // 动态调整编码器码率
        if (encoder) {
            g_object_set(G_OBJECT(encoder), 
                        "bitrate", 
                        static_cast<guint>(target_kbps), 
                        nullptr);
        }
        
        // 同时调整传输层速率（单位：Mbps）
        const uint32_t target_mbps = (target_bps + 500000) / 1000000;
        if (m_destTID > 0 && target_mbps > 0) {
            ExChangeTransSpeedOfTID(
                static_cast<DWORD>(m_destTID), 
                static_cast<DWORD>(target_mbps)
            );
        }
    }
    
    return true;
}
```

### 3.5 循环播放机制

#### 3.5.1 EOS处理
```cpp
void VideoSender::on_timer_check_appsink() {
    if (!pipeline) return;
    
    // 检查管线消息
    GstBus* bus = gst_element_get_bus(pipeline);
    while (true) {
        GstMessage* msg = gst_bus_pop_filtered(
            bus, 
            (GstMessageType)(GST_MESSAGE_ERROR | GST_MESSAGE_EOS)
        );
        
        if (!msg) break;
        
        if (GST_MESSAGE_TYPE(msg) == GST_MESSAGE_EOS) {
            // 收到EOS，回到文件开头继续播放
            gst_element_seek_simple(
                pipeline,
                GST_FORMAT_TIME,
                (GstSeekFlags)(GST_SEEK_FLAG_FLUSH | GST_SEEK_FLAG_KEY_UNIT),
                0  // 回到0位置
            );
            gst_element_set_state(pipeline, GST_STATE_PLAYING);
        } 
        else if (GST_MESSAGE_TYPE(msg) == GST_MESSAGE_ERROR) {
            GError* err = nullptr;
            gchar* debug = nullptr;
            gst_message_parse_error(msg, &err, &debug);
            std::cerr << "[VideoSender] GStreamer error: " 
                      << err->message << std::endl;
            g_error_free(err);
            g_free(debug);
        }
        
        gst_message_unref(msg);
    }
    gst_object_unref(bus);
    
    // 拉取预览帧（见3.6节）
    // ...
}
```

```cpp
void VideoSender::on_timer_check_appsink() {
    // ... EOS处理代码 ...
    
    if (!appsink_preview) return;
    
    // 尝试拉取预览帧（非阻塞）
    GstSample *sample = gst_app_sink_try_pull_sample(
        GST_APP_SINK(appsink_preview), 0);
    
    if (sample) {
        GstBuffer *buffer = gst_sample_get_buffer(sample);
        GstCaps *caps = gst_sample_get_caps(sample);
        GstStructure *structure = gst_caps_get_structure(caps, 0);
        
        // 获取图像尺寸
        int width, height;
        gst_structure_get_int(structure, "width", &width);
        gst_structure_get_int(structure, "height", &height);
        
        // 映射buffer数据
        GstMapInfo map;
        gst_buffer_map(buffer, &map, GST_MAP_READ);
        
        // 转换为OpenCV Mat（BGR格式）
        cv::Mat frame(height, width, CV_8UC3, 
                     (char*)map.data, cv::Mat::AUTO_STEP);
        
        // 深拷贝（因为buffer会被释放）
        cv::Mat frame_copy = frame.clone();
        
        gst_buffer_unmap(buffer, &map);
        gst_sample_unref(sample);
        
        // 隐藏"等待信号"提示
        QLabel *statusLabel = playerWidget->findChild<QLabel *>("statusLabel");
        if (statusLabel && statusLabel->isVisible()) {
            statusLabel->setVisible(false);
        }
        
        // 更新UI显示
        lastFrameAtMs = frameClock.elapsed();
        ShowImage(frame_copy);  // 推送到PlayerWidget
    } 
    else {
        // 无帧可用，显示等待提示
        if (lastFrameAtMs < 0) {
            QLabel *statusLabel = playerWidget->findChild<QLabel *>("statusLabel");
            if (statusLabel && !statusLabel->isVisible()) {
                statusLabel->setVisible(true);
            }
        }
    }
}
```

### 4.3 RTP数据接收与解析

#### 4.3.1 SpeedControl回调入口
```cpp
bool VideoReceiver::onSpeedControlRecvData(
    DWORD srcTID,
    const std::shared_ptr<BYTE> &pBuffer,
    DWORD dwLength,
    long int &recvtime,
    long int &fb_send_time) {
    
    if (!appsrc || !pBuffer || dwLength < 2) {
        return true;
    }
    
    m_rxRawPktsWindow += 1;  // 统计原始包数
    
    // 过滤TID（如果设置了期望TID）
    if (m_expectedRemoteTID != 0 && srcTID != m_expectedRemoteTID) {
        return true;
    }
    
    const BYTE *typed = pBuffer.get();
    
    // 检查数据类型标识（0x00表示视频RTP）
    if (typed[0] != 0x00) {
        return true;
    }
    
    // 提取RTP包
    const DWORD rtp_len = dwLength - 1;
    const BYTE *rtp = typed + 1;
    
    // 验证RTP版本（V=2, 即0x80）
    if (rtp_len < 12 || ((rtp[0] & 0xC0) != 0x80)) {
        return true;
    }
    
    // 解析RTP序列号（网络字节序）
    const uint16_t seq = (static_cast<uint16_t>(rtp[2]) << 8) 
                       | static_cast<uint16_t>(rtp[3]);
    
    // 计算丢包
    if (m_hasSeq) {
        const uint16_t delta = static_cast<uint16_t>(seq - m_lastSeq);
        
        // delta在合理范围内（避免序列号回绕误判）
        if (delta > 0 && delta < 0x8000) {
            if (delta > 1) {
                // 丢包数 = delta - 1
                m_lostPktsWindow += static_cast<uint32_t>(delta - 1);
            }
            m_lastSeq = seq;
        }
    } else {
        // 第一个包，初始化序列号
        m_hasSeq = true;
        m_lastSeq = seq;
    }
    
    // 更新统计
    m_lastRemoteTID = srcTID;
    m_recvPktsWindow += 1;
    m_recvBytesWindow += rtp_len;
    m_rxRtpPktsWindow += 1;
    m_rxRtpBytesWindow += rtp_len;
    
    // 推送到appsrc
    GstBuffer *gstbuf = gst_buffer_new_allocate(nullptr, rtp_len, nullptr);
    if (!gstbuf) {
        return true;
    }
    
    gst_buffer_fill(gstbuf, 0, rtp, rtp_len);
    
    GstFlowReturn ret = gst_app_src_push_buffer(
        GST_APP_SRC(appsrc), gstbuf);
    
    if (ret != GST_FLOW_OK) {
        std::cerr << "[VideoReceiver] appsrc push failed: " 
                  << ret << std::endl;
    }
    
    // 每秒输出统计
    const qint64 elapsedMs = m_rxStatClock.elapsed();
    if (elapsedMs >= 1000) {
        const uint64_t bps = (m_rxRtpBytesWindow * 8ULL * 1000ULL) 
                           / static_cast<uint64_t>(elapsedMs);
        const uint64_t pps = (m_rxRtpPktsWindow * 1000ULL) 
                           / static_cast<uint64_t>(elapsedMs);
        
        std::cout << "[VideoReceiver] TID=" << getTID()
                  << " from=" << srcTID
                  << " raw_pkts=" << m_rxRawPktsWindow
                  << " rtp_pps=" << pps
                  << " rtp_kbps=" << (bps / 1000ULL)
                  << std::endl;
        
        // 重置统计窗口
        m_rxRawPktsWindow = 0;
        m_rxRtpPktsWindow = 0;
        m_rxRtpBytesWindow = 0;
        m_rxStatClock.restart();
    }
    
    return true;
}
```

#### 4.3.2 NetCombTransfer回调（兼容路径）
```cpp
bool VideoReceiver::onNetCombTransferBuffer(
    DWORD srcTID,
    const std::shared_ptr<BYTE> &pBuffer,
    DWORD dwLength,
    bool bMsg,
    int64_t packet_recv_timeStamp) {
    
    // 忽略消息类型
    if (bMsg) {
        return true;
    }
    
    if (!appsrc || !pBuffer || dwLength < 2) {
        return true;
    }
    
    m_rxRawPktsWindow += 1;
    
    // TID过滤
    if (m_expectedRemoteTID != 0 && srcTID != m_expectedRemoteTID) {
        return true;
    }
    
    // 定义RTP处理函数（lambda）
    auto process_typed_rtp = [&](DWORD remoteTID, 
                                  const BYTE *typed, 
                                  DWORD typed_len) -> bool {
        if (!typed || typed_len < 2) {
            return true;
        }
        
        // 检查类型标识
        if (typed[0] != 0x00) {
            return true;
        }
        
        const DWORD rtp_len = typed_len - 1;
        const BYTE *rtp = typed + 1;
        
        // ... 序列号解析、丢包统计、推送appsrc ...
        // （与onSpeedControlRecvData相同）
        
        return true;
    };
    
    const BYTE *raw = pBuffer.get();
    
    // 尝试解析NetCombTransfer封装格式
    if (dwLength >= 21) {
        // 读取data_len字段（偏移12字节）
        const DWORD data_len = CIMsg::ReadData(
            const_cast<BYTE *>(raw + 12));
        
        // 验证长度
        if (data_len >= 2 && data_len + 20 == dwLength) {
            const BYTE *typed = raw + 20;
            
            // 检查是否为RTP包
            if (typed[0] == 0x00 && ((typed[1] & 0xC0) == 0x80)) {
                return process_typed_rtp(srcTID, typed, data_len);
            }
        }
    }
    
    // 尝试直接解析（无封装）
    if (dwLength >= 2 && raw[0] == 0x00 && ((raw[1] & 0xC0) == 0x80)) {
        return process_typed_rtp(srcTID, raw, dwLength);
    }
    
    return true;
}
```

```cpp
void VideoReceiver::on_timer_check_appsink() {
    if (!appsink) return;
    
    // 检查管线消息（错误、警告等）
    if (pipeline) {

                    std::cerr << "[VideoReceiver] Pipeline Warning: " 
                              << err->message << std::endl;
                    g_error_free(err);
                    g_free(debug);
                    break;
                }
                default:
                    break;
            }
            gst_message_unref(msg);
        }
        gst_object_unref(bus);

    // 尝试拉取解码帧（超时10ms）

        gst_buffer_map(buffer, &map, GST_MAP_READ);

        // 深拷贝
        cv::Mat frame_copy = frame.clone();
```

### 4.5 反馈机制

#### 4.5.1 反馈包生成与发送
```cpp
void VideoReceiver::on_timer_send_feedback() {
    // 检查是否有发送端TID
    if (m_lastRemoteTID == 0) {
        return;
    }
    
    // 计算统计窗口时长
    const qint64 elapsedMs = m_feedbackClock.elapsed();
    if (elapsedMs <= 0) {
        return;
    }
    
    // 计算接收速率（bps）
    const uint32_t recv_bps = static_cast<uint32_t>(
        (m_recvBytesWindow * 8ULL * 1000ULL) 
        / static_cast<uint64_t>(elapsedMs)
    );
    
    // 计算丢包率（千分比）
    const uint32_t expected_pkts = m_recvPktsWindow + m_lostPktsWindow;
    const uint32_t loss_x1000 = expected_pkts 
        ? static_cast<uint32_t>(
            (static_cast<uint64_t>(m_lostPktsWindow) * 1000ULL) 
            / expected_pkts
          )
        : 0;
    
    // 最后序列号
    const uint32_t last_seq_u32 = static_cast<uint32_t>(m_lastSeq);
    
    // 构造反馈包：[0x01][recv_bps][loss_x1000][last_seq]
    const DWORD totalLen = 1 + 4 + 4 + 4;
    std::shared_ptr<BYTE> sendBuf(new BYTE[totalLen], 
                                  std::default_delete<BYTE[]>());
    BYTE *p = sendBuf.get();
    
    p[0] = 0x01;  // 反馈包标识
    
    // 网络字节序
    uint32_t v = htonl(recv_bps);
    memcpy(p + 1, &v, 4);
    
    v = htonl(loss_x1000);
    memcpy(p + 5, &v, 4);
    
    v = htonl(last_seq_u32);
    memcpy(p + 9, &v, 4);
    
    // 发送反馈包
    SendTIDDataWithSpeedControl(
        m_lastRemoteTID,
        sendBuf,
        totalLen,
        true,   // isUrgent（反馈包优先）
        false,  // isSleep
        false   // isRBUDP（使用MRUDP）
    );
    
    // 重置统计窗口
    m_recvBytesWindow = 0;
    m_recvPktsWindow = 0;
    m_lostPktsWindow = 0;
    m_feedbackClock.restart();
}
```

#### 4.5.2 反馈包格式
```

1           4       recv_bps        接收速率（bps，网络字节序）
5           4       loss_x1000      丢包率*1000（网络字节序）
9           4       last_seq        最后序列号（网络字节序）
```

#### 5.3.4 **重要说明：视频数据传输路径**

**当前视频数据使用MRUDP传输，而非RBUDP！**

在代码中，`SendTIDDataWithSpeedControl`的最后一个参数`isRBUDP`决定使用哪个传输协议：

```cpp

    false,  // isSleep
    false   // isRBUDP = false，使用MRUDP！
);

// VideoReceiver发送反馈包

    m_lastRemoteTID, 
    sendBuf, 
    totalLen, 
    true,   // isUrgent（反馈包优先）
    false,  // isSleep
    false   // isRBUDP = false，使用MRUDP！
);
```

**传输路径确认：**
```
VideoSender/VideoReceiver
  ↓
SendTIDDataWithSpeedControl(..., isRBUDP=false)
  ↓
SpeedControlManager::DoSendData(..., isRBUDP=false)
  ↓
Term2TermTransmission::PushSendData(..., isRBUDP=false)
  ↓
DataWithPacketInfo::Encode(..., isRBUDP=false)  // 封装时标记为MRUDP
  ↓
SendTIDData(...)  // NetCombTransfer API
  ↓
CNetTerminalManager::SendData(...)
  ↓
MRUDP传输层  // 最终使用MRUDP协议发送
```

**为什么使用MRUDP而非RBUDP？**
- MRUDP提供更灵活的可靠性控制
- 支持多路径传输，提高传输稳定性
- 具有更好的拥塞控制机制
- 适合实时视频流的特性（允许部分丢包）

**RBUDP vs MRUDP：**
- RBUDP（Reliable Blast UDP）：适合大文件传输，强调可靠性
- MRUDP（Multi-path Reliable UDP）：适合实时流传输，平衡可靠性与实时性

### 5.4 传输层协议栈

#### 5.4.1 完整封装格式
```
[应用层] VideoSender/VideoReceiver
    ↓
[封装层] 0x00 + RTP包
    ↓
[SpeedControl层] 速率控制 + 统计 + DataWithPacketInfo封装
    ↓                      (isRBUDP=false标记)
[NetCombTransfer层] 20字节头 + 数据
    ↓
[MRUDP层] MRUDP头 + 数据  ← 使用MRUDP协议（非RBUDP）
    ↓
[UDP层] UDP头 + 数据
    ↓
[IP层] IP头 + 数据
```

**重要：** 虽然代码中有`isRBUDP`参数，但视频传输始终设置为`false`，表示使用MRUDP传输。

```cpp
void PlayerWidget::pushFrame2Show(const cv::Mat &mat) {
    std::lock_guard<std::mutex> guard(images_lock);
    
    // 转换并添加到队列
    images << QPixmap::fromImage(MatToQImage2(mat));
}

void PlayerWidget::paintEvent(QPaintEvent *event) {
    if (!is_play) return;
    
    QPainter painter(this);
    std::lock_guard<std::mutex> guard(images_lock);
    
    // 队列为空
    if (images.empty()) {
        // 如果有最后一帧，继续显示
        if (!lastPixmap.isNull()) {
            painter.drawPixmap(rect(), lastPixmap);
        }
        return;
    }
    
    // 显示队列头部帧
    lastPixmap = images.front();
    painter.drawPixmap(rect(), lastPixmap);
    
    // 队列管理
    if (images.size() >= 2) {
        images.pop_front();  // 弹出已显示的帧
    } else if (images.size() == 1) {
        // 只有一帧时，根据lockLastImg决定是否保留
        if (!lockLastImg) {
            images.pop_front();
        }
    }
    
    // 限制队列长度
    while (images.size() > len_buf) {
        images.pop_front();
    }
}
```

```cpp
QImage PlayerWidget::MatToQImage2(const cv::Mat &mat) {
    QImage img;
    int chana = mat.channels();
    
    if (3 == chana) {
        // 3通道：BGR或RGB
        cv::Mat tmp;
        if (is_rgb) {
            cv::cvtColor(mat, tmp, cv::COLOR_BGR2RGB);
        } else {
            tmp = mat;
        }
        img = QImage(static_cast<uchar *>(tmp.data), 
                    tmp.cols, tmp.rows, 
                    QImage::Format_RGB888).copy();
    }
    else if (4 == chana) {
        // 4通道：ARGB
        img = QImage(static_cast<uchar *>(mat.data), 
                    mat.cols, mat.rows, 
                    QImage::Format_ARGB32).copy();
    }
    else {
        // 单通道：灰度图
        img = QImage(mat.cols, mat.rows, QImage::Format_Indexed8);
        uchar *matdata = mat.data;
        for (int row = 0; row < mat.rows; ++row) {
            uchar *rowdata = img.scanLine(row);
            memcpy(rowdata, matdata, mat.cols);
            matdata += mat.cols;
        }
    }
    
    return img;
}
```

#### 7.2.3 RTP序列号处理
```cpp
// 序列号是16位无符号整数，会回绕
uint16_t seq = (rtp[2] << 8) | rtp[3];

// 计算序列号差值（处理回绕）
if (m_hasSeq) {
    uint16_t delta = seq - m_lastSeq;
    
    // delta < 0x8000 表示正常递增
    // delta >= 0x8000 表示回绕或乱序
    if (delta > 0 && delta < 0x8000) {
        if (delta > 1) {
            // 丢包数 = delta - 1
            m_lostPktsWindow += (delta - 1);
        }
        m_lastSeq = seq;
    }
}

// 示例：
// lastSeq=65534, seq=65535, delta=1 → 正常
// lastSeq=65535, seq=0, delta=1 → 回绕，正常
// lastSeq=100, seq=50, delta=65486 → 乱序，忽略
```

## 11. 总结与展望

### 11.1 当前实现总结

本视频传输模块实现了一个完整的端到端实时视频传输系统，具有以下特点：

**架构设计：**
- 采用GStreamer作为编解码引擎，成熟稳定
- 自定义传输层（SpeedControl + MRUDP），支持多路径和可靠传输
- 清晰的分层设计，各模块职责明确

**核心功能：**
- MP4文件循环播放作为视频源
- H.264编码，RTP封装
- 自适应码率控制（200kbps ~ 8Mbps）
- 实时反馈机制（200ms周期）
- 丢包检测与统计
- 本地预览与远端显示

**性能特性：**
- 零延迟配置（tune=zerolatency）
- 快速编码（speed-preset=superfast）
- 最小抖动缓冲（200ms）
- 实时统计输出（1秒周期）

**用户体验：**
- 零接触启动（接收端自动启动）
- 状态提示（等待/启动/发送中）
- 最后一帧锁定（避免黑屏闪烁）
- 循环播放（无缝衔接）

### 11.2 技术亮点

1. **深度集成设计**：GStreamer appsink/appsrc与自定义传输层无缝对接
2. **自适应传输**：基于反馈的动态码率调整
3. **多线程协作**：Qt主线程、GStreamer线程池、传输层线程协同工作
4. **内存安全**：智能指针管理、深拷贝策略、原子操作
5. **容错机制**：序列号回绕处理、网络中断恢复、EOS循环播放

### 11.3 可优化方向

#### 11.3.1 短期优化
- **硬件加速**：使用nvv4l2h264enc（NVIDIA）或vaapih264enc（Intel）替代软件编码
- **更精细的码率控制**：引入平滑算法，避免码率抖动
- **FEC前向纠错**：在高丢包场景下提升鲁棒性
- **多码率支持**：根据网络状况切换不同分辨率

#### 11.3.2 中期扩展
- **摄像头支持**：替换MP4文件源为v4l2src
- **音频传输**：添加音频编码（Opus）与同步
- **多路视频**：支持多个发送端/接收端
- **录制功能**：添加filesink分支保存视频

#### 11.3.3 长期演进
- **WebRTC集成**：使用标准WebRTC协议栈
- **SVC编码**：可伸缩视频编码，支持多层码率
- **AI增强**：超分辨率、降噪、带宽预测
- **云端部署**：支持云端转码与分发

### 11.4 已知问题与限制

1. **固定视频源**：当前只支持MP4文件，不支持实时摄像头
2. **单向传输**：只支持单向视频传输，不支持双向通话
3. **无音频**：当前只传输视频，不包含音频
4. **固定分辨率**：640x480固定分辨率，不支持动态调整
5. **软件编码**：CPU占用较高，需要硬件加速支持

### 11.5 使用建议

**开发环境：**
- 使用Ubuntu 20.04或更高版本
- 安装完整的GStreamer插件集
- 使用Qt Creator进行开发

**测试环境：**
- 至少双核CPU，4GB内存
- 千兆网络环境
- 准备好测试视频文件（testp4.mp4）

**生产部署：**
- 启用硬件编码器
- 调整网络缓冲区大小
- 配置防火墙规则
- 监控CPU和网络使用率
