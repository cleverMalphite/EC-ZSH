# 弱网环境视频传输优化方案 (Plan)

## 1. 目标
针对应急场景下的弱网环境（低带宽、高丢包、高抖动），通过收敛反馈路径、平滑发送节奏、以及多级自适应降级，提升视频传输的流畅度与抗灾能力。

## 2. 现有问题分析
*   **反馈空窗与耦合：** 接收端通过 200ms 一次的 `0x01` 包反馈应用层 EMA 统计，与底层的 BBR 反馈并存，导致发送端响应慢且逻辑冲突。
*   **I帧码率冲击：** 固定 I 帧周期产生周期性突发流量，在弱网下极易诱发拥塞。
*   **调节维度单一：** 仅调节 Bitrate，未在极端带宽下利用分辨率和帧率的弹性。
*   **队列积压：** 应用层码率调整不及时，导致数据在 `SpeedControl` 队列中积压，产生额外延迟。

## 3. 优化实施步骤

### 阶段一：编码参数深度调优 (Smooth Encoding)
*   [ ] **引入 Intra-refresh：** 修改 `VideoSender.cpp`，移除 `key-int-max`，启用 `intra-refresh=true`（帧内刷新），消除 I 帧爆发。
*   [ ] **调优 VBV Buffering：** 显式设置 `vbv-bufsize`，配合 `byte-stream=true`，确保码率输出严格平滑。
*   [ ] **降低预设等级：** 统一使用 `speed-preset=ultrafast` 降低编码端延迟。

### 阶段二：接收端逻辑瘦身与反馈收敛 (Receiver Slimming)
*   [ ] **移除冗余反馈：** 删除接收端原有的应用层 EMA 统计和 `0x01` 反馈包发送逻辑。
*   [ ] **保留本地抖动缓冲区：** 确保动态抖动缓冲区（Jitter Buffer）逻辑不受影响，维持播放平滑性。
*   [ ] **统一大脑：** 修改 `VideoSender`，使其不再监听 `0x01` 消息，改为通过 API 直接轮询或订阅 `Term2TermQos` 模块计算出的带宽和 RTT。

### 阶段三：实现多级自适应控制器 (Multi-stage Scaling)
*   [ ] **设计状态机：** 根据底层反馈的带宽定义三种工作模式：
    *   **Normal:** 640x480, 30fps, 1Mbps+
    *   **Low Bandwidth:** 480x360, 20fps, 500kbps - 1Mbps
    *   **Extreme:** 320x240, 15fps, < 500kbps
*   [ ] **引入队列负反馈：** 增加对 `SpeedControl` 待发送队列深度的监控，若队列超过阈值，强制执行降级或跳帧。

### 阶段四：算法平滑与稳定性优化 (Stability)
*   [ ] **带宽估计平滑：** 对底层反馈的带宽数据引入一阶低通滤波，防止编码器码率频繁震荡。
*   [ ] **拥塞/丢包区分判断：** 结合 RTT 变化趋势优化降速权重。

## 4. 验证指标
*   **端到端延迟：** 预期在 5% 丢包环境下，平均延迟下降 100ms 以上（得益于响应加快和队列积压减少）。
*   **画面平滑度：** 观察播放端，消除因 I 帧丢失导致的周期性花屏。
*   **抗拥塞能力：** 在带宽剧烈波动时，系统能迅速平滑切换分辨率，不出现长时间卡顿。
